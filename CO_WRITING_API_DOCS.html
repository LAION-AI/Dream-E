Dream-E Co-Writing Mode

API Reference & Feature Documentation — v2026.03

Co-Writing Mode is a structured storytelling workspace in Dream-E designed for authors who want to plan, outline, and develop a narrative before (or alongside) building the interactive game graph. It provides dedicated node types for story structure—story root, plot arcs, acts, and character relationship webs—plus an AI chat agent that can manipulate all of these via a programmatic API.

How it differs from Game Mode

AspectGame ModeCo-Writing Mode
Primary purpose Build playable interactive fiction Outline, develop, and structure a narrative
Canvas Single canvas with all node types Dual canvas system (Story + Character)
Unique nodes Scene, Choice, Modifier, Comment Story Root, Plot, Act, Character (plus Scene & Comment)
Edge types Standard directed flow edges Relationship edges with metadata (type, beginning, act developments, ending)
AI commands Scene/entity/variable CRUD All game-mode commands + 17 co-write commands
Player mode Full adventure engine playback N/A (narrative planning tool)

The Dual Canvas System

In co-write mode, a tab bar appears above the React Flow canvas with two tabs:

  • Story Canvas — Shows scene, comment, storyRoot, plot, and act nodes. This is the narrative structure view where you lay out the overall story arc and plot lines.
  • Character Canvas — Shows character nodes only. These are visual character cards linked to entities. Relationship edges between characters display on this canvas, letting you map out the character web with detailed relationship data.

Edges are filtered so that only edges whose both endpoints are visible on the current canvas are shown. This prevents dangling connections from appearing when the other node lives on the other canvas tab.

Toolbar behavior: In co-write mode, the toolbar adapts to the active canvas. On the Story Canvas you can drag out Story Root (max 1), Plot Arc, Act, Scene, and Comment nodes. On the Character Canvas you can drag out Character nodes only.

2. Node Types

Story Root Node

The central anchor of a co-writing project. There is exactly one Story Root node per project. It holds the high-level story metadata that defines the narrative's foundation: title, genre, target audience, logline, characters, and synopsis. Plot nodes auto-connect to it.

StoryRootNodeData Fields

title string
The working title of the story. Can be changed at any time.
genre string
Primary genre (e.g., Fantasy, Sci-Fi, Romance, Thriller, Horror, Mystery, Literary Fiction). Sets reader expectations for tone, pacing, and story conventions.
targetAudience string
Who the story is written for (e.g., Young Adult, Adult, Middle Grade, New Adult). Affects language complexity, theme maturity, and content boundaries.
punchline string
The "logline" or elevator pitch — capture the entire story in 1–2 sentences. Formula: [Character] must [Goal] before [Stakes/Deadline], but [Obstacle].
mainCharacter { name: string, role: string }
The protagonist — the character whose journey drives the story. Should have a clear want (external goal) and a need (internal growth).
antagonist { name: string, role: string }
The primary opposing force. Can be a villain, rival, nature, society, or the protagonist's own flaws.
supportingCharacters Array<{ name: string, archetype: string, customArchetype?: string }>
Characters who help, hinder, or reflect the protagonist. Each should serve a narrative function and feel like a real person.
protagonistGoal string
The one clear, concrete objective the protagonist pursues. This launches the plot (Act 1) and gets resolved at the climax (Act 3). Should be specific and measurable.
summary string
A 300–500 word overview of the entire story from beginning to end, including the ending. Covers: hook, rising action, midpoint shift, crisis/climax, and resolution.
image string?
Optional mood/concept image for the story. Can be AI-generated via the generate_node_image command.

Visual Appearance

Rendered with a purple accent border and a crown/book icon. The title is displayed prominently. If an image is set, it appears as a background or card header. Only one Story Root is permitted per project.

How to Create

Drag the "Story Root" item from the toolbar on the Story Canvas. If one already exists, the toolbar item is hidden. Programmatically, the root is created automatically when a co-write project is initialized; use update_story_root to populate its fields.

Plot Node

Represents a narrative arc or storyline thread. Each plot node auto-connects to the Story Root when created, establishing a parent-child relationship. A project can have any number of plots.

PlotNodeData Fields

name string
The plot's display name (e.g., "The Ring Quest", "Forbidden Love").
plotType string
One of: Main Plot, Relationship Plot, Antagonist Plot, Character Development Plot, Subplot, Custom.
customPlotType string?
A free-text label used when plotType is "Custom" (e.g., "Political Intrigue Thread").
description string
Description of the plot's arc — what it covers, how it escalates, how it resolves.
image string?
Optional mood/concept image for the plot arc. AI-generatable.

Plot Types Explained

TypeDescription
Main PlotThe central storyline driven by the protagonist's primary goal. Every other plot connects to or contrasts with this one. Follows: Goal → Obstacles → Escalation → Crisis → Climax → Resolution.
Relationship PlotA storyline about a key relationship (romantic, familial, or friendship). Provides the emotional core and runs parallel to the main plot.
Antagonist PlotThe storyline from the antagonist's perspective. Should escalate in parallel, creating increasing pressure on the protagonist.
Character Development PlotAn internal arc where a character overcomes a flaw, learns a truth, or transforms. The "need" vs "want" dynamic.
SubplotA secondary storyline that enriches the world or themes. Good subplots echo the main plot's themes in a different key.
CustomAny arc that doesn't fit standard categories: mystery threads, political intrigue, quests-within-quests, etc.

Visual Appearance

Rendered with an orange accent border. Displays the plot name and type badge. Auto-connected to the Story Root via a directed edge upon creation.

How to Create

Drag from toolbar, or use the create_plot API command. The edge to the Story Root is automatically created.

Act Node

Represents a major structural division of the story (e.g., Act 1: Setup, Act 2: Confrontation, Act 3: Resolution). Acts define the timeline and pacing structure. They can be connected to Plot nodes via relationship edges to specify which parts of each plot play out in each act.

ActNodeData Fields

actNumber number
The act number (e.g., 1, 2, 3). Used for ordering and display.
name string
Display name of the act (e.g., "The Setup", "Rising Action", "The Reckoning").
description string
Description of what happens during this act — key events, turning points, character states.
image string?
Optional mood/concept image for this act. AI-generatable.

Three-Act Structure Reference

ActProportionPurpose
Act 1 — Setup~25%Establish the ordinary world, introduce characters, present the inciting incident that disrupts the status quo.
Act 2 — Confrontation~50%Rising action, escalating conflicts, midpoint shift, complications, character growth under pressure.
Act 3 — Resolution~25%Crisis, climax, resolution. The protagonist faces the final challenge and the story's central question is answered.
Tip: You can use more than three acts for finer-grained pacing. Each act should end with a major turning point that propels the story forward.

Visual Appearance

Rendered with a teal accent border. Shows "Act N" with the act name and description excerpt. Sorted by actNumber in the context listing.

Character Node

A visual card on the Character Canvas that links to an entity in the project's entity system. Character nodes allow you to map out relationships between characters using specialized relationship edges. Changes sync bidirectionally with the entity system.

CharacterNodeData Fields

entityId string
The ID of the linked entity. The character node is a visual proxy — all actual data (name, description, profile, image) lives on the entity.

Predefined Profile Fields

The Character Node Inspector surfaces these predefined profile fields (stored on the entity's profile object):

FieldTypeDescription
characterTypeDropdownNarrative archetype: Protagonist, Antagonist, Sidekick, Mentor, Love Interest, Rival, Comic Relief, Guardian, Herald, Trickster, Shapeshifter, Threshold Guardian, Anti-hero, Foil, Custom
ageTextCharacter's age or age range (e.g., "34", "Late twenties", "Ancient")
genderTextGender identity, informing pronoun usage and social dynamics
appearanceTextareaPhysical description: height, build, hair, eyes, distinguishing features, typical clothing. Used by AI for image generation consistency
occupationTextProfession or role in the story world (e.g., "Blacksmith", "Court Spy", "Starship Engineer")
problemSolvingStrategiesTextareaHow the character approaches problems: fight, negotiate, trick, analyze, avoid? Defines their agency in scenes

Below the predefined fields, a ProfileViewer component allows adding arbitrary free-form profile fields. Any key-value pair can be added (e.g., "motivations", "speech_style", "inventory"). Profile values are inline-editable (click to edit).

Entity Features Available on Character Nodes

  • Reference Image — A portrait that can be uploaded, AI-generated, or selected from project assets. Used by the image generator for visual consistency.
  • Reference Voice — An audio clip for TTS voice matching. A 10–30 second clip with clear speech works best. Can be uploaded or AI-generated.

Scene Node

Scene nodes are still available on the Story Canvas in co-write mode. They work identically to game mode: background image, story text, speaker name, choices, linked entities, and media (background music, voiceover). Scenes allow you to draft actual content alongside the structural planning.

Comment Node

Comments remain available on the Story Canvas for designer notes, TODO markers, and organizational annotations. They do not affect gameplay or narrative structure. Each comment has a configurable background color for visual categorization.

3. Edge / Relationship Types

Character-to-Character Relationships

When you draw an edge between two Character Nodes on the Character Canvas, it is created as a relationship edge with a specialized data payload. These edges are rendered with a dashed pink bezier curve and a label showing the relationship type.

RelationshipEdgeData Fields (Character-Character)

relationshipType string
The nature of the relationship (e.g., "Allies", "Rivals", "Romantic", "Mentor-Student", "Siblings", "Enemies").
description string
Freeform description of the relationship's dynamics, power balance, and significance.
status string
Current status of the relationship (e.g., "Strong", "Strained", "Broken", "Evolving").
history string
Background history of how the relationship formed and key past events.
beginning string?
How the relationship exists at the very start of the story. The baseline the audience measures all future changes against.
actDevelopments Array<{ actLabel: string, development: string }>?
Act-by-act tracking of how the relationship evolves. Each entry maps an act label to a description of the relationship's shift during that act. Relationships should change — new information, betrayals, shared ordeals, and revelations all reshape how characters feel about each other.
ending string?
How the relationship stands at the end of the story. Has it deepened, broken, reversed, or reached a new understanding? Should feel like a natural consequence of all preceding events.
entityId string?
If a dedicated "relationship entity" was created in the entity system, its ID is stored here.

Act-to-Plot Connections

Relationship edges can also connect Act Nodes to Plot Nodes. These edges use the plotInvolvement field to describe which parts of a plot arc play out during a specific act.

RelationshipEdgeData Fields (Act-Plot)

plotInvolvement string?
What parts of the plot play out in this act. For example: "Act 1 introduces the mystery and plants the first clues" or "Act 2 deepens the romantic subplot with a betrayal revelation." Mapping plots to acts helps visualize pacing — if all plot developments are crammed into one act, the others may feel thin.
Pacing check: Connecting acts to plots creates a visual matrix of your story's pacing. Every plot should have meaningful development in at least two acts. If one act has no plot connections, consider whether it's carrying enough narrative weight.

Story Root-to-Plot Links

When a Plot Node is created (via toolbar drag or API), an edge is automatically created from the Story Root to the new plot node. These are standard directed edges (not relationship edges) representing the parent-child hierarchy: the Story Root is the parent of all plot arcs.

4. Co-Writing API Command Reference

All commands use the Dream-E command block format. The AI chat agent emits these inline in its responses, and they are executed sequentially by the agentic loop. 

<<<SW_CMD:command_name>>>
{"param": "value", "anotherParam": 42}
<<</SW_CMD>>>

The cowrite group contains 17 commands organized into six sub-groups. All commands are available when the project is in co-write mode.

Update any combination of story root fields. All parameters are optional — only the fields you include will be changed.

ParameterTypeRequiredDescription

5. Context System

What the AI Sees

Every message to the AI chat agent is prefixed with a [Current Game State] block generated by getGameContext(). This snapshot includes:

  • Project info: Title, mode (cowrite or game), start node ID
  • Scenes: All scene nodes with IDs, labels, text preview (120 chars), choices, connections, media indicators
  • Entities: All entities with IDs, categories, names, image/profile indicators, summary preview
  • Variables: All global variables with IDs, names, types, default values
  • Edges: All edges with IDs, source/target, source handles

Co-Write Context Extension

When the project is in co-write mode, a [CO-WRITING STRUCTURE] section is appended containing:

  • Story Root: Node ID, title, genre, target audience, punchline, main character, antagonist, supporting characters, protagonist goal, summary (200-char preview), image indicator
  • Plots: Each plot with ID, name, type, image indicator, description preview (100 chars)
  • Acts: Each act with ID, act number, name, description preview (100 chars), sorted by act number
  • Character Nodes: Each character node with ID and linked entity name
  • Relationships: Each relationship edge with ID, source/target IDs, type, description preview (80 chars)

The Agentic Loop

The AI operates in a multi-turn agentic loop managed by aiChatService.ts:

  1. User sends a message (prefixed with fresh game state)
  2. Agent responds with text and <<<SW_CMD:...>>> command blocks
  3. Commands are extracted and executed sequentially via executeCommand()
  4. Results (successes, errors, JSON data for query commands) are sent back to the agent with updated game state
  5. Agent reviews results, can retry failures or issue additional commands
  6. Loop continues until the agent responds with no commands (task complete) or MAX_ITERATIONS is reached
MAX_ITERATIONS = 10 per user message. This allows up to ~50+ commands across multiple turns, enough to build a substantial story structure in one conversation step. The limit prevents runaway loops.

Data Commands

The following co-write commands are classified as "data commands" — their full JSON response is included in the results message sent back to the agent (not just a success/fail indicator):

  • get_story_root
  • list_plots
  • list_acts
  • list_relationships

6. Image Generation

ImageGenerationOverlay Features

The ImageGenerationOverlay is a shared modal component used across the app for AI image generation. It can be opened from scene inspectors, entity managers, character inspectors, story root, plot, and act inspectors.

UI Elements

  • Provider info bar: Shows the active AI provider and model name with a green "active" indicator
  • Prompt textarea: Free-text description of the desired image
  • Aspect ratio selector: 7 preset ratios (see table below)
  • Reference images: Upload from disk or select from project assets. Displayed as 64px thumbnail previews with remove buttons
  • Generate button: Triggers generation with loading spinner state
  • Result preview: Shows the generated image with "Use This Image" and "Regenerate" options
  • Error display: Shows API errors with warning icon

Character Archetypes

The following character types are available in the Character Type dropdown. Each comes with narrative expectations that help maintain consistency:

ArchetypeDescription
ProtagonistThe central character whose journey drives the story. Must have a clear external goal (what they want) and an internal need (what they must learn or overcome).
AntagonistThe primary force opposing the protagonist. Best antagonists have understandable motivations and serve as a dark mirror reflecting what the protagonist could become.
SidekickLoyal companion who provides humor, practical skills, or emotional grounding. (e.g., Samwise Gamgee, Ron Weasley)
MentorWise guide who teaches essential skills or truths. Often removed to force the hero to act independently. (e.g., Gandalf, Obi-Wan)

RivalCompetitor who pushes the protagonist to grow. Shares the protagonist's goals but competes for the same prize.
Comic ReliefLightens the mood with humor. Best comic relief characters have genuine emotional depth beneath the jokes.
GuardianProtector who shields the protagonist at great personal cost. Represents safety and stability.
HeraldAnnounces change and kicks off the adventure. Delivers the "call to adventure" that disrupts the ordinary world.
TricksterClever, morally ambiguous disruptor who uses wit, deception, or chaos. Challenges authority and rigid thinking.

Plot Type Explanations

TypeExplanation
Main PlotCentral storyline driven by the protagonist's primary goal. Follows: Goal → Obstacles → Escalation → Crisis → Climax → Resolution. Every other plot connects to or contrasts with this one.
Relationship PlotAbout a key relationship (romantic, familial, friendship). Runs parallel to the main plot and often provides the emotional core.
Antagonist PlotTold from the antagonist's perspective. Should escalate in parallel, creating increasing pressure. Best stories show the antagonist as a mirror of the protagonist.
Character Development PlotInternal arc: overcoming a flaw, learning a truth, or transforming. The "need" vs "want" — what the character thinks they need vs what they actually need to grow.
SubplotSecondary storyline enriching the world or themes. Good subplots echo the main plot's themes in a different key.
CustomAny narrative thread that doesn't fit standard categories: mystery threads, political intrigue, quests-within-quests.

Three-Act Structure

The classic story architecture dividing a narrative into three major phases:

ActName~ProportionKey Elements
Act 1 Setup ~25% Ordinary World → Inciting Incident → First Plot Point
Establish characters, setting, and status quo. The inciting incident disrupts everything. By the end of Act 1, the protagonist commits to the journey (or is forced into it).
Act 2 Confrontation ~50% Rising Action → Midpoint → Complications → Second Plot Point
The protagonist faces escalating obstacles. The midpoint shifts from reactive to proactive (or vice versa). Subplots deepen. Stakes rise. By the end, all seems lost.
Act 3 Resolution ~25% Crisis → Climax → Resolution
The protagonist faces the final challenge, drawing on everything they've learned. The central question is answered. Loose ends are tied. The new status quo is established.

Logline / Punchline Formula

The logline formula captures your entire story in one to two sentences:

// The formula:
[Character] must [Goal] before [Stakes/Deadline], but [Obstacle].

// Example:
"A blind mapmaker must chart a vanishing continent before it
disappears forever, but every map she draws erases part of her memory."

// Components:
Character  = Who (with a defining trait that creates tension)
Goal      = What they must accomplish (specific, measurable)
Stakes    = What happens if they fail (time pressure adds urgency)
Obstacle  = What makes success difficult (ideally connected to the character)

Relationship Development

Relationships are the emotional engine of stories. Dream-E's co-write mode tracks relationship evolution across three dimensions:

  • Beginning: How the relationship exists at the story's start. The baseline the audience measures all changes against. A strong beginning makes every subsequent shift feel earned.
  • Act Developments: Track how the relationship evolves through each act. Each act should bring at least one significant shift — new information, betrayals, shared ordeals, and revelations all reshape dynamics.
  • Ending: How the relationship stands at the story's conclusion. Has it deepened, broken, reversed, or reached a new understanding? Should feel like a natural consequence of everything that happened — surprising yet inevitable.
