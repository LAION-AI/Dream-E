Game State API Reference

56+ commands for complete control of Dream-E game state. Used by the AI chat agent and exposed on window.storyAPI for console testing.

Architecture

Files:
  gameStateAPI.ts         — Command handlers (executeCommand entry point)
  gameStateAPI.registry.ts — Declarative command metadata (auto-generates system prompt)
  gameStateAPI.context.ts  — Serializes game state for AI context
  gameStateAPI.types.ts    — APIResult, ValidationError, CommandMeta types

Invocation:
  AI chat agent emits:  <<<SW_CMD>>> command_name { "param": "value" }
  aiChatService.ts parses and calls:  executeCommand("command_name", { param: "value" })
  Handler validates, executes, returns:  { success: true, ...data } | { success: false, error, suggestion }

Error handling:
  Every validation error includes a `suggestion` field with available IDs/values.
  Example: "Entity not found: entity_xyz. Available entities: entity_abc (Elena), entity_def (Forest)"

AI Chat Agentic Loop

File: aiChatService.ts

User sends message
  → buildUserMessage(text) — wraps with game state context
  → POST /api/chat (Claude Code subprocess)
  → Parse response for <<<SW_CMD>>> blocks
  → executeCommand() for each command
  → buildResultsMessage(results) — full JSON for data commands, status for others
  → Feed results back to agent
  → Loop until no commands or MAX_ITERATIONS (10)
  → Return final text response
System prompt auto-generation: The AI chat system prompt is auto-generated from the command registry metadata (gameStateAPI.registry.ts). This means the system prompt always matches the available commands with zero drift.

Console Access

All commands are exposed on window.storyAPI in the browser console:

// Example console usage:
window.storyAPI.executeCommand('list_scenes', {})
window.storyAPI.executeCommand('create_entity', { category: 'character', name: 'Elena' })
window.storyAPI.executeCommand('search_music', { query: 'tavern cozy medieval' })
