AI Settings & Configuration

Store: useImageGenStore.ts | UI: AISettingsModal.tsx

Overview

The AI Settings modal (gear icon in editor toolbar) configures three AI subsystems:

  1. Image Generation — Scene backgrounds, entity portraits
  2. Text-to-Speech (TTS) — Voiceover narration
  3. Story Writer — Open World mode AI scene generation

All settings persist to localStorage via Zustand's persist middleware (store version 4).

Google API Key

A single Google API key is shared by three services when their provider is set to Gemini:

  • Gemini image generation
  • Gemini TTS
  • Gemini story writer (Open World mode)

Field: googleApiKey. Entered once in the AI Settings modal.

Default Image Style

Field: defaultImageStyle. A text string appended to ALL image generation prompts. Example: "digital fantasy art, vibrant colors, dramatic lighting". Overrides auto-detected art style.

Reference Images

For Gemini provider only: entity reference images are sent alongside the image prompt as inlineData with the instruction "Using the reference images above for visual consistency, generate a new image: [prompt]". This provides visual consistency for character appearances across scenes.

Chunked Streaming

TTS uses chunked pseudo-streaming for low latency:

  • Chunk 1: ~100 characters (arrives fast)
  • Chunk 2: ~200 characters
  • Chunk 3+: ~400 characters each
  • All chunks sent in parallel to POST /api/generate-tts
  • Delivered in order as they complete (not FIFO)
  • Concatenated into single audio file for storage

Generate Voiceover Button

In SceneInspector Media tab: "Generate Voiceover" button. Disabled if no story text or voiceover already attached. Uses the scene's storyText as input to the TTS model.

System Prompt (Editable)

The default system prompt defines the AI's role, output format, and behavioral constraints. Key sections:

  • Role: Game master/narrator for open world interactive fiction
  • Writing style: Screenplay-inspired, punchy prose, vivid descriptions, natural dialogue
  • Analysis protocol: Fill analysis fields (playerGoalHypothesis, etc.) BEFORE writing scene
  • Entity update protocol: MANDATORY profilePatch + stateChanges for any entity state change
  • State change detail: Use specific, vivid descriptions (not generic "affected by magic")
  • Output format: Structured JSON with all required fields

Writer Instruction (Editable)

Appended to the end of the user message (after all context sections). Reinforces key requirements:

  • 100-300 word continuation based on player action
  • Analysis fields filled first (drive the scene)
  • Entity profile updates MANDATORY for persistent changes
  • Specific wording for effects (exact terms, how manifested)

Max Context Tokens

Configurable in AI Settings. Controls when context compression kicks in. For Gemini models with 1M+ context windows, 500K is a reasonable default. Compression strategy:

  1. Drop full profiles of non-scene entities
  2. Compress oldest story scenes to summaries (keep at least 5 recent in full)

API keys are stored in localStorage. They are never sent to any server other than the configured AI provider endpoints. The Vite dev server acts as a proxy but the API keys are sent from the browser directly in the request body.

Exported Defaults

For programmatic access or reset functionality:

  • DEFAULT_WRITER_SYSTEM_PROMPT — exported from useImageGenStore.ts
  • DEFAULT_WRITER_INSTRUCTION — exported from useImageGenStore.ts
  • Reset buttons in AI Settings modal restore these defaults
