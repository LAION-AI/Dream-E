Node Types

Scene Node (Blue)

The primary content node. Displays story text to the player during gameplay.

AspectDetail
ColorBlue (node-scene token)
Size200px min-width, 280px max-width
Input handleTop center (target)
Output handlesOne per choice, right side (source)
HeaderFilm icon + label. Green dot if start node.
BodyThumbnail (lazy-loaded via IntersectionObserver) + story text preview (80 chars max)
FooterChoice labels with output handles

Choice Node (Yellow)

Conditional branching. Routes the player based on a variable condition.

AspectDetail
ColorYellow (node-choice)
Input handleTop center
Output handlesBottom-left "True" (green), bottom-right "False" (red)
DisplayCondition expression in monospace (e.g., HP >= 50)

Choice Node Data

  • condition.variableA — Variable name to check
  • condition.operator — One of: >, <, =, !=, >=, <=, contains
  • condition.valueB — Comparison value (number or string)
  • condition.useVariable — If true, valueB references another variable

Modifier Node (Green)

Changes a variable's value when the player passes through it.

AspectDetail
ColorGreen (node-modifier)
Input/OutputTop center / bottom center (single path)
ModesMath (add/sub/mul/div), Set (direct assign), Random (range)
DisplayOperation preview in monospace (e.g., HP = HP + 5)

Comment Node (Gray, Dashed)

Designer annotation. Not connected to flow, not executed. Color-customizable.

AspectDetail
BorderDashed, gray (or custom color from 8 presets)
HandlesNone (not part of story flow)
DisplayText preview (150 chars max)

Inspector Panel (Right Sidebar)

Context-sensitive property editor. Width: 25vw (min 380px, max 600px). Glassmorphism design with color-coded left border.

Scene Inspector — 4 Tabs

Media Tab

  • Background Image — upload or drag-drop, displays thumbnail
  • Background Music — upload audio file, "Keep playing in next scene" checkbox
  • Voiceover — upload audio or click "Generate Voiceover" (Gemini TTS), "Auto-play on enter" checkbox

Content Tab

  • Speaker Name — optional character name (e.g., "Narrator", "Guard")
  • Story Text — large textarea, supports {{variableName}} substitution
  • View AI Response — opens modal to view/edit raw AI JSON (OW-generated scenes only)

Outputs Tab

  • Choices list — add, edit labels, reorder, delete choices
  • Each choice creates an output handle on the node
  • Connect choice handles to target nodes by dragging
  • Conditions can be set per choice (variable-based visibility)

World Tab

  • Entity linkers for 4 categories (Characters, Locations, Objects, Concepts)
  • Linked entities shown as chips with amber dot if they have situational state text
  • Click entity to open SceneEntityOverlay — per-entity text area for situational attributes
  • Add entities from dropdown list

Top Section (Above Tabs)

  • Node label input
  • Start Node indicator (green pulsing dot) + "Set as Start Node" button (only one allowed)
  • "Play from Here" — starts game at this scene with fresh variables
  • "Continue from Here" — starts game at this scene with autosaved variables
  • Open World Mode toggle (checkbox)

Choice Inspector

  • Node label input
  • Variable selector dropdown (or quick-create inline)
  • Operator dropdown (>, <, =, !=, >=, <=, contains)
  • Value input (number or string)

Modifier Inspector

  • Node label input
  • Target variable dropdown (or quick-create inline)
  • Mode tabs: Math (4 operation buttons + value), Set (text input), Random (min/max)
  • Logic preview in monospace box

Comment Inspector

  • Label input
  • Note textarea (resizable, 150px min-height)
  • Color picker: 8 preset squares (gray, red, orange, yellow, green, blue, purple, pink)

Entity System

Categories

  • Characters — NPCs, party members, antagonists
  • Locations — Places, buildings, regions
  • Objects — Items, artifacts, weapons
  • Concepts — Factions, magic systems, lore

Entity Fields

FieldPurpose
nameDisplay name
descriptionDetailed text (used in AI context)
summaryBrief 1-2 sentence overview
profileStructured key-value data (appearance, personality, stats, etc.). Inline-editable.
referenceImagePortrait/visual reference. Sent to Gemini for image gen consistency.
stateHistoryChronological log of state changes across scenes (magical effects, injuries, relationship shifts)

State Change History

Each EntityStateChangeEvent records:

  • sceneId + sceneLabel — Which scene the change occurred in
  • sceneSummary — What happened in that scene
  • timestamp — When it occurred
  • playerAction — What the player did to trigger it
  • stateChanges — Array of specific change descriptions (verbatim, detailed)

Entity Linking

Entities are linked to scenes via the World tab in SceneInspector. Linked entity IDs are stored in linkedCharacters, linkedLocations, linkedObjects, linkedConcepts arrays on the scene node data. These determine which entities appear in the AI context and which reference images are sent for image generation.

Variable System

Variable Types

  • integer — Whole numbers (HP, gold)
  • float — Decimal numbers (multipliers)
  • boolean — True/false flags
  • string — Text values (status effects)
  • collection — String arrays (inventory)

HUD Display

Variables with showInHUD: true appear as progress bars in the top-left HUD during gameplay. Configurable: icon, color, min/max values.

Templates

Pre-defined variable sets: RPG Basic Stats (HP, MP, Strength, etc.), Currency (Gold), Inventory.

Import / Export

Export

  1. Rehydrates blob URLs back to base64 (rehydrateForSave())
  2. Serializes project as JSON
  3. Wraps in ZIP file (.dream-e.zip)
  4. Downloads via File System Access API or classic download

Import

  1. User selects .zip file
  2. projectsDB.importProject(file) extracts JSON
  3. Saves to IndexedDB with new ID
  4. Navigates to imported project

Undo / Redo

AspectDetail
Max history8 snapshots (reduced from 50 for memory)
Debounce500ms for text changes, immediate for structural (add/delete node/edge)
Lean snapshotsBinary assets stripped (backgroundImage, backgroundMusic, voiceoverAudio, referenceImage, aiResponse, constructedContext) → replaced with '__asset_stripped__'
RehydrationOn undo/redo, assets copied from current live project back into restored snapshot
SuppressionBatch operations (OW scene creation) use suppressHistoryRecording() to avoid recording intermediate states

Auto-Layout

Bottom-left button arranges all nodes by BFS depth from start node. Column spacing: 400px, row spacing: 250px. All connections preserved.

Focus-on-Return

When returning from play mode to editor, useEditorStore.focusNodeId is set to the last played scene. Editor uses useReactFlow().setCenter() to pan/zoom to that node. Falls back to findDeepestSceneNode() (BFS from start node).
