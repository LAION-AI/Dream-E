Stores & Type Definitions

useProjectStore (Main Project Store)

File: src/stores/useProjectStore.ts (~1,282 lines)

Manages the currently loaded project and all editing operations. Uses Zustand with Immer middleware for immutable state updates.

State

{
  currentProject: Project | null     // The open project
  isDirty: boolean                   // Unsaved changes flag
  selectedNodeId: string | null      // Currently selected node
  selectedEdgeId: string | null      // Currently selected edge
  isLoading: boolean                 // Loading from IndexedDB
  isSaving: boolean                  // Saving to IndexedDB
  error: string | null               // Error state
  history: Project[]                 // Undo/redo stack (MAX=8, lean snapshots)
  historyIndex: number               // Current position in history
}

Key Actions

ActionDescription
loadProject(id)Load from IndexedDB, initialize optional fields (entities, notes, assetNames, etc.)
saveProject()Rehydrate blob URLs → base64, save to IndexedDB
saveAndOffloadAssets()Save, then convert base64 → blob URLs for memory optimization
closeProject()Revoke all blob URLs, clear state
addNode(node)Add node to graph (immediate history)
updateNode(id, updates)Update node data (debounced history)
deleteNode(id)Delete node + connected edges (immediate history)
deleteNodes(ids)Batch delete (single history entry)
addEdge(edge)Create connection
deleteEdge(id)Remove connection
addEntity(entity)Create entity (character/location/object/concept)
updateEntity(id, updates)Update entity (Immer Object.assign)
deleteEntity(id)Remove entity + unlink from all scenes
updateEntityState(nodeId, entityId, text)Set per-scene entity state note on node.data.entityStates
addVariable() / updateVariable() / deleteVariable()Variable CRUD
updateNotes(text)Update project notes (debounced)
undo() / redo()Navigate history with asset rehydration
addChatMessage() / clearChatMessages()AI chat history management

History System Details

  • MAX_HISTORY = 8 — Reduced from 50 for memory (combined with lean snapshots)
  • HISTORY_DEBOUNCE_DELAY = 500ms — Text changes are debounced
  • Immediate recording — Structural changes (add/delete node/edge) record immediately
  • Lean snapshots — Binary assets + aiResponse + constructedContext stripped
  • Rehydration on undo/redo — Assets copied from live project to restored snapshot
  • suppressHistoryRecording() — Returns unsuppress function. Used for batch OW operations.

useEditorStore (Canvas UI State)

File: src/stores/useEditorStore.ts

State

{
  viewport: { x: number, y: number, zoom: number }
  activeTool: 'select' | 'pan' | 'connect'
  draggingNodeType: string | null
  panels: {
    toolbar: boolean     // Left sidebar
    inspector: boolean   // Right panel
    minimap: boolean     // Bottom-right minimap
  }
  preferences: {
    showGrid: boolean
    snapToGrid: boolean
    gridSize: number          // Default 20
    showLabels: boolean
    animateEdges: boolean
    autoSaveInterval: number  // Seconds
  }
  clipboard: { nodes: StoryNode[], edges: StoryEdge[] } | null
  contextMenuOpen: boolean
  contextMenuPosition: { x: number, y: number }
  lastMousePosition: { x: number, y: number }
  pendingUploads: number      // File uploads in progress
  focusNodeId: string | null  // Node to center on when entering editor
}

Persistence

Only preferences and panels persist to localStorage. Viewport, clipboard, and transient UI state are session-only.

useImageGenStore (AI Settings)

File: src/stores/useImageGenStore.ts

State

{
  // Image Generation
  provider: 'bfl' | 'openai-compatible' | 'gemini'
  apiKey: string              // BFL or OpenAI key
  model: string               // Provider-specific model name
  endpoint: string            // API base URL (OpenAI-compatible)
  defaultImageStyle: string   // Appended to all image prompts

  // Google (shared)
  googleApiKey: string        // Used by Gemini image, TTS, and writer
  geminiImageModel: string    // Gemini image generation model

  // TTS
  tts: {
    enabled: boolean
    model: string             // e.g., 'gemini-2.5-flash-preview-tts'
    voice: string             // e.g., 'Zephyr'
    instruction: string       // Narration style direction
  }

  // Story Writer (Open World)
  writer: {
    provider: 'gemini' | 'openai-compatible'
    model: string             // e.g., 'gemini-3-flash-preview'
    endpoint: string
    apiKey: string
    systemPrompt: string      // Editable, resettable to DEFAULT_WRITER_SYSTEM_PROMPT
    instruction: string       // Editable, resettable to DEFAULT_WRITER_INSTRUCTION
    maxContextTokens: number  // Default 500,000
  }

  // ASR (Speech-to-Text)
  asr: {
    enabled: boolean
    model: string
    deviceId: string
  }
}

Persistence

All settings persist to localStorage. Store version: 4. Migration from v3 adds maxContextTokens to writer settings.

Core Type Definitions

Project (types/project.ts)

interface Project {
  id: string;
  info: ProjectInfo;              // title, author, description, cover, theme, dates
  settings: ProjectSettings;      // startNodeId, HUD config, audio, text speed, save slots
  nodes: StoryNode[];             // All editor nodes (union type)
  edges: StoryEdge[];             // All connections
  globalVariables: Variable[];    // Game stats and flags
  entities: Entity[];             // Characters, locations, objects, concepts
  notes: string;                  // Project-wide notes (freeform text)
  assetNames: string[];           // Named media references
  chatMessages: ChatMessage[];    // AI chat history
}

Entity (types/project.ts)

interface Entity {
  id: string;                                    // "entity_{uuid}"
  category: 'character' | 'location' | 'object' | 'concept';
  name: string;
  description: string;
  summary?: string;                              // Brief 1-2 sentence overview
  profile: Record<string, unknown>;              // Structured key-value data (editable)
  referenceImage?: string;                       // Base64 or blob URL (portrait/visual)
  referenceVoice?: string;                       // Voice sample
  defaultMusic?: string;                         // Default music track
  stateHistory?: EntityStateChangeEvent[];       // Chronological state changes
  linkedCharacters?: string[];                   // Related character IDs
  linkedLocations?: string[];                    // Related location IDs
  linkedObjects?: string[];                      // Related object IDs
  linkedConcepts?: string[];                     // Related concept IDs
  createdAt?: number;
  updatedAt?: number;
}

EntityStateChangeEvent (types/project.ts)

interface EntityStateChangeEvent {
  sceneId: string;               // Which scene the change occurred in
  sceneLabel?: string;           // Human-readable scene name
  sceneSummary: string;          // What happened in that scene
  timestamp?: number;            // When it occurred (Date.now())
  playerAction?: string;         // What the player did to trigger it
  stateChanges: string[];        // Array of specific change descriptions (verbatim)
}

type StoryNode = SceneNode | ChoiceNode | ModifierNode | CommentNode;

// Each has: id, type, position: {x, y}, label, data: { ... type-specific fields }

SceneDisplayData (types/gamestate.ts)

interface SceneDisplayData {
  nodeId: string;
  label: string;
  speakerName?: string;
  text: string;                    // Interpolated story text
  choices: PlayerChoice[];         // Resolved choices with conditions
  backgroundImage?: string;       // Base64 or blob URL
  backgroundMusic?: string;       // Base64 or blob URL

Naming inconsistency: SceneNode data uses voiceoverAudio but SceneDisplayData (runtime) uses voiceover. These are the same concept with different field names.

Data Flow: Store Interactions

Editor Mode:
  User → Inspector UI → useProjectStore.updateNode() → Zustand/Immer → React Flow re-render

Play Mode (Standard):
  usePlayerStore.startGame(project)
    → Snapshot project into player store
    → Initialize session with variables from project.globalVariables
    → Navigate to startNodeId
    → setCurrentScene() with getBlobUrl() for assets

Play Mode (Open World):
  Player action
    → useProjectStore.getState().currentProject (FRESH, not player store snapshot)
    → buildOpenWorldContext(project, session, action)
    → POST /api/open-world → streaming AI response
    → parseOpenWorldResponse() → OpenWorldResult
    → onComplete callback updates UI with pending result
    → handleAdvanceToOwScene():
      → useProjectStore.addNode() + addEdge()
      → Deferred: useProjectStore.updateEntity() for each entityUpdate
      → Deferred: saveAndOffloadAssets()
