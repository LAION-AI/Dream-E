Overview

The Game State API is the bridge between the AI chat agent and the Dream-E editor. Every modification to the project — creating scenes, connecting nodes, generating images, managing entities — goes through this API.

Scene Nodes (Blue)

Display story content and wait for player input. Each scene has text, choices, media, and entity links.

  • backgroundImage — scene backdrop
  • backgroundMusic — audio track
  • musicKeepPlaying — continue to next scene
  • voiceoverAudio — one-shot narration
  • voiceoverAutoplay — auto-play on load
  • speakerName — who is speaking
  • storyText — the narrative text
  • choices[] — player choice buttons
  • linkedCharacters/Locations/Objects/Concepts
  • entityStates — per-entity scene context

Entities

Characters, locations, objects, and concepts. The world-building database.

  • name — display name
  • category — character | location | object | concept
  • description — full details
  • summary — 100-200 word brief
  • referenceImage — visual reference
  • referenceVoice — voice identity clip
  • defaultMusic — auto-play on encounter
  • musicFadeIn / musicFadeOut — ms durations

Modifier Nodes (Green)

Invisible. Change variable values during gameplay. Three modes:

  • Math — add, subtract, multiply, divide
  • Set — assign a value directly
  • Random — generate random number in range

Branch Nodes (Yellow)

Invisible. Route the flow based on a condition.

  • Evaluates: variableA operator valueB
  • TRUE → "success" output
  • FALSE → "failure" output
  • Operators: > < = != >= <= contains

Data Model

Dream-E projects contain:

  • Nodes — scenes, modifiers, branches, comments (placed on canvas)
  • Edges — directed connections between nodes (the story flow)
  • Entities — characters, locations, objects, concepts (world-building)
  • Variables — game state tracking (HP, gold, flags, inventory)

Command Format

The AI agent embeds commands inline in its response using this block syntax:

<<<SW_CMD:action_name>>>
{"param1": "value1", "param2": 42}
<<</SW_CMD>>>

Multiple commands can appear in one response. They execute sequentially in order.

Success Response

{ "success": true, "nodeId": "node_abc123", ... }

Error Response

{
  "success": false,
  "error": "Scene not found: node_xyz",
  "suggestion": "Available scenes: node_a1b2 (\"Forest\"), node_c3d4 (\"Cave\")"
}

Browser Console API

All commands are exposed on window.storyAPI for testing and debugging:

// Create a scene
await storyAPI.create_scene({label: "Forest", storyText: "You enter a dark forest..."})
// → {success: true, nodeId: "node_abc123", choiceIds: []}

// Try an invalid command — see error + suggestion
await storyAPI.update_scene({sceneId: "nonexistent"})
// → {success: false, error: "Scene not found: nonexistent", suggestion: "Available scenes: ..."}

// Generate an image
await storyAPI.generate_scene_image({sceneId: "node_abc123", prompt: "dark enchanted forest"})
// → {success: true, sceneId: "node_abc123", message: "Image generated..."}

// List everything
await storyAPI.list_scenes()
await storyAPI.list_entities()
await storyAPI.list_variables()

Connection Commands 3

Manage directed edges between nodes. Edges define the story flow. Scene node edges use choice IDs as the sourceHandle.

Parameters

patch_entity_profileApply JSON Patch operations to an entity's profileWRITE

Applies a list of JSON Patch (RFC 6902 subset) operations to the entity's profile dictionary. This allows selective updates without rewriting the entire profile. Supports nested paths with / separators.

Parameters

NameTypeRequiredDescription
entityIdstringrequiredEntity ID
operationsarrayrequiredArray of patch operations (see below)

Supported Operations

OpPathValueDescription
addrequiredrequiredAdd or set a key. Creates intermediate objects automatically. For arrays, use /- to append.
replacerequiredrequiredReplace an existing key's value. Fails if the path doesn't exist.
removerequired—Delete a key from the profile. Fails if the path doesn't exist.

Path Syntax

  • /key — top-level field (e.g., /appearance)
  • /key/subkey — nested field (e.g., /relationships/rival)
  • /key/0 — array index (e.g., /abilities/0 = first ability)
  • /key/- — append to array (e.g., /abilities/-)

Returns

{success: true, appliedOps: number, profile: object}

Asserts: Entity must exist. Profile must already exist (use set_entity_profile first). Each operation must have a valid op, path, and (for add/replace) value. The replace and remove ops fail if the target path doesn't exist.

Example — Update Relationships and Add an Ability

<<<SW_CMD:patch_entity_profile>>>
{
  "entityId": "entity_blackbeard",
  "operations": [
    {"op": "replace", "path": "/relationships/rival", "value": "Captain Whitmore — now allied after the Battle of Coral Bay"},
    {"op": "add", "path": "/relationships/newAlly", "value": "The Siren Queen — uneasy alliance"},
    {"op": "add", "path": "/abilities/-", "value": "Gunpowder alchemy"},
    {"op": "replace", "path": "/goals", "value": "Defend Coral Bay from the Imperial fleet"},
    {"op": "remove", "path": "/flaws"}
  ]
}
<<</SW_CMD>>>

Example — Add a New Nested Section

<<<SW_CMD:patch_entity_profile>>>
{
  "entityId": "entity_blackbeard",
  "operations": [
    {"op": "add", "path": "/inventory", "value": {"weapons": ["Cutlass", "Flintlock"], "treasures": ["Ruby pendant"]}}
  ]
}
<<</SW_CMD>>>

Image Generation Configuration

Image generation uses a server-side API endpoint (POST /api/generate-image). The provider, API key, model, and endpoint are configured in the AI Settings panel (gear icon in the toolbar).

Supported Providers

Black Forest Labs (BFL)

FLUX image generation models. Default provider.

  • Endpoint: https://api.bfl.ai/v1/{model}
  • Auth: X-Key header
  • Default model: flux-pro-1.1
  • Flow: Submit → Poll → Download
  • Models: flux-pro-1.1, flux-dev, flux-schnell, etc.

OpenAI-Compatible

Any API that implements the OpenAI images endpoint.

  • Endpoint: {baseUrl}/images/generations
  • Auth: Bearer token
  • Default model: dall-e-3
  • Flow: Single POST → Response
  • Providers: OpenAI, Together, Replicate, etc.

Open World Mode

Open World Mode allows players to type free-form actions during gameplay instead of choosing from pre-defined options. The AI generates new scene content, images, and choices on the fly.

How It Works

  1. Player enables "Open World Mode" checkbox in the Scene Inspector
  2. Clicks "Start Here" or "Continue Here" to enter the player
  3. A text input appears below the dialog box
  4. Player types what they want to do and presses Enter
  5. The AI generates a continuation, image, and new choices
  6. A golden "Continue to Next Scene" button appears when ready

Context Construction (Token Budget System)

Image Generation

  • Image prompt is generated by the AI based on entity profiles and scene content
  • Up to 3 retries on failure (content filter, API error)
  • Each retry progressively simplifies the prompt
  • If all retries fail, scene continues without an image

Content Guidelines

The AI follows adult-movie-level content guidelines: violence, romance, and mature themes are acceptable. Extreme content is rephrased to stay appropriate. The AI never breaks character or refuses outright — it finds creative redirections.
