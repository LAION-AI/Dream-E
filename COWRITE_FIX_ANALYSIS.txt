PHASE 1: PROBLEM IDENTIFICATION
===========================================================================

SYMPTOM: The AI ignores co-write rules — it executes commands in its first
response instead of discussing the story with the user first. It should
ask questions, make proposals, and wait for confirmation before executing
any <<<SW_CMD:...>>> commands.

ROOT CAUSE: The system prompt is far too long. The co-write preamble
structure is:

  1. [~350 words] Critical rules (Rule 1-6)
  2. [~50 words]  "Your Role" section (first copy)
  3. [~5234 words] CHARACTER_DEPTH_GUIDE.md (entire guide!)
  4. [~50 words]  "Your Role" section (DUPLICATE - second copy)
  5. [~200 words] Co-Write Data Model
  6. [~300 words] Mandatory Workflow Order
  7. [~300 words] Confirmation Protocol
  8. [~200 words] Walking-Through Mode
  9. [~150 words] Teaching Mode
  10. [~100 words] Context Awareness
  11. [~150 words] Co-Write Scene Nodes
  12. [~80 words]  Command Format + Agentic Loop
  13. [~60 words]  Important Rules
  14. [~500 words] Command Reference

  TOTAL: ~7,724 words / ~50,000 characters in the system prompt

ISSUES IDENTIFIED:

1. GUIDE PLACEMENT BURIES RULES
   The 5234-word Character Depth Guide is placed BETWEEN the critical rules
   (at the top) and the operational instructions (workflow, confirmation
   protocol, data model). By the time the model reads through the entire
   guide, it has largely "forgotten" the critical rules. LLMs attend more
   strongly to the beginning and end of the context, not the middle
   (the "lost in the middle" problem, documented by Liu et al. 2023).

2. DUPLICATED "YOUR ROLE" SECTION
   Lines 983-989 and 996-1001 in gameStateAPI.registry.ts contain the
   exact same "Your Role" text. The first copy appears before the guide,
   the second after. This wastes ~50 words and signals confusion in the
   prompt structure.

3. SYSTEM PROMPT TOO LONG FOR RELIABLE RULE COMPLIANCE
   Research and practice show that shorter system prompts with clear
   structure yield much more reliable instruction following. A system prompt
   of ~650 words (rules + workflow + commands) would be far more effective
   than one with ~7700 words where the rules are 5000 words away from the
   commands.

4. GUIDE IS REFERENCE MATERIAL, NOT INSTRUCTIONS
   The CHARACTER_DEPTH_GUIDE is reference material that the AI should
   *apply* when writing characters. It doesn't need to be in the system
   prompt — it works just as well (or better) in the user message context,
   where it appears alongside the actual project state the AI is working
   with.
